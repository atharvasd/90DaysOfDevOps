# Lab 02: Terraform Import Block (Declarative — v1.5+)

**Topic:** Modern Terraform — Bringing existing resources under management

---

## Overview

When infrastructure already exists (created manually via Console or CLI), you need to "import" it into Terraform state so Terraform can manage it going forward. Terraform 1.5+ introduced the declarative `import` block which is much safer than the legacy CLI approach.

### Key Concepts
- **`terraform import` (Legacy CLI)** — Requires you to manually write the full resource config first, then run `terraform import` to add it to state. Error-prone.
- **`import` block (Modern, v1.5+)** — You declare the import in HCL, run `terraform plan`, and Terraform generates the resource config for you. Safe because you see a plan before anything changes.
- **`-generate-config-out`** — A flag on `terraform plan` that auto-generates resource blocks for imported resources. Eliminates manual config writing.
- **State** — Terraform's internal database mapping HCL config to real cloud resources. Import adds entries to state.

## 🧩 Hands-on Tasks

### Task 1: Create Infrastructure Manually (Simulating Pre-Existing)

```bash
# Create a GCS bucket via CLI (not Terraform)
gcloud storage buckets create gs://ace-import-demo-<yourname> \
    --location=us-east1 \
    --uniform-bucket-level-access

# Create a Compute Engine firewall rule via CLI
gcloud compute firewall-rules create legacy-allow-http \
    --network=default \
    --allow=tcp:80 \
    --target-tags=web
```

### Task 2: Write Import Blocks (Modern Approach)

```hcl
# main.tf
terraform {
  required_version = ">= 1.5.0"
  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 6.0"
    }
  }
}

provider "google" {
  project = "<your-project-id>"
  region  = "us-east1"
}

# Import block for the GCS bucket
import {
  to = google_storage_bucket.imported_bucket
  id = "<your-project-id>/ace-import-demo-<yourname>"
}

# Import block for the firewall rule
import {
  to = google_compute_firewall.imported_firewall
  id = "projects/<your-project-id>/global/firewalls/legacy-allow-http"
}
```

### Task 3: Generate Config Automatically

```bash
# Initialize Terraform
terraform init

# Generate resource configs from the import blocks
terraform plan -generate-config-out=generated_imports.tf
```

This creates `generated_imports.tf` with the complete resource blocks matching reality:

```hcl
# generated_imports.tf (auto-generated by Terraform)
resource "google_storage_bucket" "imported_bucket" {
  name                        = "ace-import-demo-<yourname>"
  location                    = "US-EAST1"
  force_destroy               = false
  uniform_bucket_level_access = true
  # ... all other attributes filled in automatically
}

resource "google_compute_firewall" "imported_firewall" {
  name    = "legacy-allow-http"
  network = "default"
  allow {
    protocol = "tcp"
    ports    = ["80"]
  }
  target_tags = ["web"]
  # ... all other attributes filled in automatically
}
```

### Task 4: Apply to Sync State

```bash
# Review the plan (should show "import" actions, NOT "create")
terraform plan

# Apply — this imports into state without modifying real infrastructure
terraform apply
```

### Task 5: Verify Import

```bash
# List all managed resources
terraform state list
# Output:
# google_storage_bucket.imported_bucket
# google_compute_firewall.imported_firewall

# Show details of imported resource
terraform state show google_storage_bucket.imported_bucket

# Confirm no drift — should show "No changes"
terraform plan
```

### Comparison: Legacy vs Modern Import

| Feature | Legacy `terraform import` | Modern `import` block |
|---|---|---|
| Config writing | Manual (you write it first) | Auto-generated |
| Plan preview | ❌ No — directly mutates state | ✅ Yes — shows plan first |
| Multiple resources | One command per resource | All in one `terraform plan` |
| Version required | Any | Terraform ≥ 1.5.0 |
| Safety | Low (can break state) | High (reversible plan) |
