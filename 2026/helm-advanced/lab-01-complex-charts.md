# Lab 01: Writing Complex Helm Charts (Subcharts and Helpers)

**Topic:** Advanced Helm

---

## Overview

Simple Helm charts just template a Deployment and a Service. But enterprise applications often consist of a frontend, a backend, and a database. Instead of putting everything in one massive chart, Helm allows you to create **Subcharts** (dependencies) and use advanced templating functions in `_helpers.tpl`.

---

## 🛠️ Hands-on Tasks

### Task 1: Create a Parent Chart with Subcharts

1. **Create the main chart:**
```bash
helm create my-enterprise-app
cd my-enterprise-app
# Delete the default templates to start clean
rm -rf templates/*
```

2. **Add a dependency:**
Open `Chart.yaml` and add a database dependency (e.g., PostgreSQL from Bitnami).
```yaml
dependencies:
  - name: postgresql
    version: 13.2.0
    repository: https://charts.bitnami.com/bitnami
```

3. **Download the dependency:**
```bash
helm dependency update
```
*Look inside the `charts/` folder. You will see a `postgresql-13.2.0.tgz` archive.*

### Task 2: Advanced `_helpers.tpl` Logic

The `templates/_helpers.tpl` file contains reusable Go template snippets.

1. **Create `templates/_helpers.tpl`:**
```go
{{/*
Generate a standard set of labels to be used across all resources
*/}}
{{- define "my-app.labels" -}}
app.kubernetes.io/name: {{ .Chart.Name }}
app.kubernetes.io/instance: {{ .Release.Name }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end -}}
```

2. **Use the helper in a ConfigMap (`templates/configmap.yaml`):**
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-config
  # Include the labels generated by the helper
  labels:
    {{- include "my-app.labels" . | nindent 4 }}
data:
  database-host: {{ .Release.Name }}-postgresql
```

### Task 3: Overriding Subchart Values

You can configure the PostgreSQL subchart directly from the parent's `values.yaml`.

Open `values.yaml` and append:
```yaml
# Note how the key 'postgresql' matches the name of the dependency in Chart.yaml
postgresql:
  auth:
    postgresPassword: "supersecretpassword"
  primary:
    persistence:
      size: 10Gi
```

### Task 4: Template and Verify

Instead of installing, use `helm template` to view the generated YAML.

```bash
helm template test-release .
```
*You will see the YAML for your ConfigMap, AND all the YAML generated by the PostgreSQL subchart, customized with your values.*

---

## ✅ Best Practices
- **Use `nindent` instead of `indent`:** In `_helpers.tpl` includes, always use `nindent` (which adds a newline before indenting). This prevents frustrating YAML formatting errors.
- **Pin Subchart Versions:** Always use exact versions for dependencies in `Chart.yaml` (e.g., `13.2.0`, not `~13.2.0`) to ensure your deployments are reproducible.
