# Lab 02: Creating Custom Ansible Modules

**Topic:** Advanced Ansible

---

## Overview

Ansible comes with thousands of modules (`apt`, `copy`, `user`, `docker_container`), but occasionally you need to automate an internal, proprietary system or a legacy API that doesn't have an official module. 

Ansible modules are just scripts (usually written in Python) that read JSON from standard input, do some work, and output JSON to standard output.

---

## 🛠️ Hands-on Tasks

### Task 1: Directory Structure

By default, Ansible looks for custom modules in a `library/` directory adjacent to your playbook.

```bash
mkdir custom-module-demo && cd custom-module-demo
mkdir library
```

### Task 2: Write the Python Module

We will write a simple module that checks if an internal "Dummy API" is reachable and returns a greeting.

Create `library/dummy_api.py`:

```python
#!/usr/bin/python

from ansible.module_utils.basic import AnsibleModule

def run_module():
    # 1. Define the module arguments
    module_args = dict(
        name=dict(type='str', required=True),
        fail_mode=dict(type='bool', required=False, default=False)
    )

    # 2. Instantiate the module
    module = AnsibleModule(
        argument_spec=module_args,
        supports_check_mode=True
    )

    # 3. Handle check mode (Dry run)
    if module.check_mode:
        module.exit_json(changed=False, msg="Check mode: Would have called the dummy API.")

    # 4. Extract variables
    name = module.params['name']
    fail_mode = module.params['fail_mode']

    # 5. Core logic
    if fail_mode:
        module.fail_json(msg="The Dummy API rejected the request!", name=name)

    # Success output
    result = dict(
        changed=True,
        greeting=f"Hello {name}, the Dummy API is responding successfully!"
    )

    module.exit_json(**result)

def main():
    run_module()

if __name__ == '__main__':
    main()
```

### Task 3: Use the Custom Module

Now we'll write a playbook to call our new `dummy_api` module.

Create `playbook.yml`:

```yaml
---
- name: Test Custom Module
  hosts: localhost
  gather_facts: no
  tasks:
    - name: Call the Dummy API normally
      dummy_api:
        name: "DevOps Engineer"
      register: api_result

    - name: Print the result
      debug:
        var: api_result.greeting

    # Uncomment this block to see how the module handles failures
    # - name: Call the Dummy API in fail mode
    #   dummy_api:
    #     name: "Bad Actor"
    #     fail_mode: true
```

### Task 4: Run the Playbook

```bash
ansible-playbook playbook.yml
```

You should see your custom python script execute, and the `debug` module will print the JSON response generated by `module.exit_json()`.

---

## ✅ Best Practices
- **Idempotency:** A good Ansible module is idempotent. If the system is already in the desired state, it should do nothing and return `changed=False`.
- **Use `AnsibleModule`:** Always import and use the `AnsibleModule` helper class from `ansible.module_utils.basic`. It handles input parsing and JSON serialization safely.
